= Panoply — Quick Start for Developers
:toc:
:toclevels: 2

== Read this first

The substantive developer education is the xref:../guide/for-developers.adoc[Developer
guide] — the Core/projection/evidence/manifest/backend model, the obligations
you sign up to, and where each mechanism will be built. This page is just the
mechanical entry point.

== Honest status

Design phase: there is no compiler or test suite to run yet. The mechanisms named
by the xref:../architecture/DESIGN-DISCIPLINE.adoc[charter] are *sketch areas*
(README + manifest) under `src/` — see xref:../architecture/ARCHITECTURE.adoc[Architecture]
for the component map. Contributions right now are *specifications, sketches, and
proofs*, not feature code against a running compiler.

== Before a PR

* New normative behaviour traces to a charter clause, or amends it via an ADR
  (`docs/decisions/`) — never weakens it silently in code.
* No axiomatic escapes in load-bearing tracks (no `believe_me`, `Admitted`,
  `unsafeCoerce`); see `.github/copilot-instructions.md`.
* Run the local gates in `scripts/` before pushing — root shape,
  AsciiDoc-only docs, and the banned-language scan.
