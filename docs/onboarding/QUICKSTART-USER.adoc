= Panoply — Quick Start for Users
:toc:
:toclevels: 2

== What is Panoply?

An *envelope-first language discipline*: every safety claim is explicit, scoped,
inspectable, and mechanically accountable. A Panoply program doesn't just
"typecheck" — it projects into a checked Core, carries evidence for its claims,
and emits a *safety-envelope manifest* listing which guarantees it earned and
which it did not.

== There is nothing to run yet — and that's stated honestly

Panoply is at the *design* phase. There is no compiler, no manifest emitter, and
no backend to target today. So this is not a "5-minute path to working software";
it is the path to *understanding* what Panoply will give you and what its claims
will mean.

== Your 5-minute path to understanding

. Read the xref:../guide/for-users.adoc[User guide] — what a manifest tells you,
  and what a "guarantee" means here (scoped, backend-relative, never global).
. Skim the xref:../wikis/GLOSSARY.adoc[Glossary] — Core, evidence, envelope,
  manifest, backend contract.
. If you want the full picture, read the
  xref:../architecture/DESIGN-DISCIPLINE.adoc[Design Discipline] (the charter).

== The one rule to remember

[quote]
____
No guarantee without an envelope. No envelope without evidence. No composition
without a composition rule.
____

If a guarantee you care about is not named in a program's manifest, treat it as
*not provided* — that is the system working as designed.
